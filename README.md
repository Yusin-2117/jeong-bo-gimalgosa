## 네트워크의 이해
- 교과서 12P
### 네트워크 장비 장비의 종류와 역할
---
- > **허브 및 스위치**
    - 허브는 **여러 컴퓨터를 하나의 네트워크로 연결**하는 장치이고, 스위치는 **필요한 장치에만 데이터를 전송**하는 장치이다.
    - >주변 장치를 연결하여 하나의 네트워크를 만들고,
    그 안에서 데이터를 전송할 수 있게 하는 장비이다.
---
- > **라우터**
    - **IP 주소를 이용해 서로 다른 네트워크를 연결하고 최적의 경로로 전달**하는 장치이다.
    - >네트워크와 네트워크를 연결하는 장비로, 서로 다른 네트워크 사이에 위치하여 데이터를 전송할 수 있게한다. 또한 외부 네트워크로 데이터가 전송되는 경로를 지정하는 역할도 한다.
---
- > **무선 AP(엑세스 포인트)**
    - **유선 네트워크를 무선(Wi-Fi) 네트워크로 연결해 주는 장치**이다.
    - >유선 랜과 무선 랜을 연결하는 장치로 와이파이 기술을 이용하여 인터넷을 사용할 수 있도록 한다.
---
- > **방화벽**
    - **방화벽은 네트워크 보안 장치**, 허용된 통신만 통과시키고 나머지는 차단한다.
    - > 외부로부터의 불법 접근, 악의적인 공격을 막아서 보안을 유지하는 기술이다.

### 무선 네트워크의 종류와 특징

| 기술              | 의미                                  | 특징                                                | 예시                    |
| --------------- | ----------------------------------- | --------------------------------------------------------------- | ------------------------ |
| **RFID**        | 전파를 이용한 **전자태그 인식 기술**              | 소형 반도체 칩이 내장된 태그나 라벨, 카드 등에 저장된 데이터를 무선 주파수로 읽어내는 인식 시스템을 말한다                     | 교통카드, 출입증, 물류 관리, 도서 관리  |
| **Wi-Fi**       | 무선 LAN 기술                           | **가까운** 거리에서 고속 무선 통신을 제공하며 인터넷 접속에 사용됨, 저렴하고 빠름                   | 가정·학교·카페 무선 인터넷          |
| **Bluetooth**   | 근거리 무선 통신 기술                        | **10m 이내** 거리에서 와이파이보다 느리게 기기 간 페어링을 데이터를 주고받음                    | 무선 이어폰, 키보드, 마우스, 스마트워치  |
| **NFC**         | 근거리 무선 통신(Near Field Communication) | **약 10cm 이내**의 매우 가까운 거리에서 빠르고 안전하게 데이터를 교환함                    | 모바일 결제, 교통카드, 출입 인증      |
| **디스플레이 공유 기술** | 화면을 다른 기기에 무선으로 전송하는 기술             | 스마트폰이나 PC의 화면을 TV·프로젝터 등에 무선으로 공유함 | 스마트 TV 미러링, 무선 발표, 화면 공유 |

### 문자열 압축
- > 런- 렝스 인코딩
     - **동일한 문자의 반복**을 반복문자 갯수, 반복 문자 쌍으로 표현하는 인코딩 알고리즘이다. 단순 반복 문자가 길 때 효율적이다.
- > 렘펠-지브 인코딩
     - **문자열에 포함된 같은 패턴의 상대적 위치와 패턴의 길이로 문자열을 치환**하여 압축하는 방법이다. 반복 패턴이 많이 포함된 문자열에 효율적이다


### 이미지 압축

| 형식            | 압축 방식    | 지원 색상    | 파일 크기 | 특징            |
| ------------- | -------- | ------- | ----- | ------------- |
| **BMP**       | 비압축 | 트루 컬러(24비트)   | 매우 큼  | 원본 이미지 저장     |
| **PNG**       | 비손실   | 트루 컬러(24비트)      | 보통    | 반복되는 색을 적은 비트로 줄여서 표현    |
| **GIF**       | 손실, 비손실    | 256색(8비트) | 작음    | 지원 색상이 256색, 지원 색보다 많으면 손실 | 
| **JPEG(JPG)** | 손실     | 트루 컬러(24비트)   | 작음    | 육안 인식 불가의 정보를 제거            |

### WAV 파일
- 아날로그 신호를 디지털로 변환 과정

| 단계                       | 설명                                                                         |
| ------------------------ | -------------------------------------------------------------------------- |
| **1. 표본화**     | 아날로그 소리 파형을 **일정한 시간 간격으로 나누어** 샘플 신호를 측정한다. |
| **2. 양자화** | 측정한 소리의 크기를 **가장 가까운 디지털 값**으로 변환한다. |
| **3. 부호화**     | 양자화된 값을 **0과 1의 이진수**로 변환하여 파일에 저장한다.            |

### 소리 압축 파일

| 형식    | 압축 방식           | 특징                                       |
| -------- | --------------- | ---------------------------------------- |
| **FLAC** | **비손실** | 음질 손실 없이 파일 크기를 줄이며, 원본 음질을 그대로 복원할 수 있음 |
| **MP3**  | **손실**       | 사람이 잘 듣지 못하는 소리를 제거하여 파일 크기를 크게 줄임       |
| **AAC**  | **손실**       | MP3보다 압축 효율이 높아 같은 용량에서 더 좋은 음질을 제공함     |

### **7비트** 아스키코드 변환
- ***7비트 → 2⁷ =128가지***
- 초기 컴퓨터와 통신 장비에서는 영문 대소문자, 숫자, 특수문자 정도만 있으면 충분했기 때문에 128개의 문자 집합이면 대부분의 용도에 맞았다.

| 문자 | 10진수 | 7비트 이진수   |
| -- | ---: | :-------- |
| A  |   65 | `1000001` |
| B  |   66 | `1000010` |
| a  |   97 | `1100001` |
| 0  |   48 | `0110000` |
| 공백 |   32 | `0100000` |

- **10진수 → 2진수 변환 과정 (65의 예)**

| 2로 나누기 |  몫 |   **나머지** |
| ------ | -: | ----: |
| 65 ÷ 2 | 32 | **1** |
| 32 ÷ 2 | 16 | **0** |
| 16 ÷ 2 |  8 | **0** |
| 8 ÷ 2  |  4 | **0** |
| 4 ÷ 2  |  2 | **0** |
| 2 ÷ 2  |  1 | **0** |
| 1 ÷ 2  |  0 | **1** |

- 이것을 위에서부터 읽으면 `100001`
